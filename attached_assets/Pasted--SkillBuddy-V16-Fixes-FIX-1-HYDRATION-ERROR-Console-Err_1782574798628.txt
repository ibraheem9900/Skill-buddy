## FIX 1 — HYDRATION ERROR (Console Error Fix)

**Error:** `Hydration failed because the server rendered HTML didn't match the client`

**Root cause:** Code uses `typeof window !== 'undefined'` or `window.innerWidth` during SSR render. This causes server/client mismatch.

**Fix — Replace ALL browser-only checks with proper SSR-safe hooks:**

```tsx
// ❌ WRONG — causes hydration mismatch:
const isMobile = window.innerWidth < 768
const isMobile = typeof window !== 'undefined' && window.innerWidth < 768

// ✅ CORRECT — SSR safe:
const [isMobile, setIsMobile] = useState(false) // false = server default

useEffect(() => {
  // Only runs on client, after hydration
  const check = () => setIsMobile(window.innerWidth < 768)
  check()
  window.addEventListener('resize', check)
  return () => window.removeEventListener('resize', check)
}, [])
```

**Also fix in Navbar — the style mismatch:**
```tsx
// ❌ WRONG — mixing inline style values that differ between server/client:
style={{ position: 'fixed', top: 0, left: 0, right: 0, zIndex: 9999 }}

// ✅ CORRECT — use className with CSS instead of inline styles for fixed positioning:
// Move all navbar positioning to CSS class, not inline style
// This prevents the server/client style string mismatch
```

```css
/* In global CSS: */
.navbar {
  position: fixed !important;
  top: 0 !important;
  left: 0 !important;

}
```

**Remove ALL inline position styles from the nav element — use only the className.**

## FIX 2 — MOBILE: ALL SECTIONS VISIBLE

**Problem:** Mobile shows only search bar, all other sections hidden. Large blank space.

**The real issue:** Sections have `min-height: 100vh` which pushes them off screen, AND `overflow: hidden` clips their content.

```css
/* GLOBAL — Apply to index.css or styles.css */
@media (max-width: 768px) {
  /* Remove forced full-viewport heights */
  section,
  .section,
  [class*="section"],
  [class*="Scene"],
  [class*="scene-"],
  [class*="snap-"] {
    height: auto !important;
    min-height: auto !important;
    max-height: none !important;
    overflow: visible !important;
  }

  /* All content visible */
  * {
    visibility: visible !important;
    opacity: 1 !important;
  }

  /* Sections not using scroll-snap on mobile */
  html, body {
    scroll-snap-type: none !important;
  }

  /* All sections stack naturally */
  main, .main-content, #root > div > main, #root > div > div:not(nav) {
    display: block !important;
    height: auto !important;
    overflow: visible !important;
  }
}
```

## FIX 3 — HAMBURGER MENU: Opens BACK Instead of Right Drawer

**Problem:** Clicking hamburger navigates back to previous page instead of opening menu.

**Root cause:** The hamburger button is wrapped in a `<Link>` or `<a>` tag, causing navigation instead of toggle.

**Fix:**
```tsx
// ❌ WRONG:
<Link to="/">
  <button onClick={() => setOpen(true)}>☰</button>
</Link>

// ❌ ALSO WRONG — button inside a tag:
<a href="/">
  <button>☰</button>
</a>

// ✅ CORRECT:
<button
  type="button"  // IMPORTANT: prevents form submission
  onClick={(e) => {
    e.preventDefault()  // IMPORTANT: prevents any navigation
    e.stopPropagation() // IMPORTANT: prevents event bubbling
    setOpen(prev => !prev)
  }}
  aria-label="Toggle menu"
  style={{ background: 'none', border: 'none', cursor: 'pointer', padding: '6px', width: 36, height: 36 }}
>
  {/* hamburger lines */}
</button>
```

**Also make sure the mobile drawer does NOT navigate on open:**
