## 10. LANGUAGE TRANSLATION — Fix Service Names & Banner Text

**Current Problem:** When language is changed, most of the site translates BUT:
- Service names (e.g., "Deep House Cleaning", "Pet Grooming", etc.) stay in English
- Banner/promotional text stays in English

**Required Fix:**

**Service names must be translated:**
- Every service name, service description, category name, and subcategory name must be added to the translation JSON files
- Wrap ALL service/category text in `t()` function
- In the data files (`src/lib/services.ts`, `src/lib/categories.ts`), instead of hardcoded English strings, use translation keys:

```typescript
// Instead of:
{ name: "Deep House Cleaning", description: "Thorough cleaning..." }

// Use translation keys:
{ nameKey: "services.deep_house_cleaning.name", descKey: "services.deep_house_cleaning.desc" }

// Then in the component:
<h3>{t(service.nameKey)}</h3>
<p>{t(service.descKey)}</p>
```

- Add ALL service name keys and their translations to all 5 language files:
  - `en.json`, `et.json`, `ru.json`, `lv.json`, `lt.json`

**Banner/promotional text must be translated:**
- Every banner title, subtitle, CTA button text, promotional offer text must be wrapped in `t()`
- Add these keys to all 5 translation files
- Special offers banners, hero banner, any promotional section text — ALL must translate

**Goal:** When user selects Estonian (or any language), literally ZERO English text should remain visible on the entire website — every single word must be in the selected language.
