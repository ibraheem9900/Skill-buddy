## 2. MOBILE: ANIMATIONS NOT WORKING

Framer Motion `whileInView` animations are not triggering on mobile. Fix:

- Add `viewport={{ once: true, amount: 0.1 }}` to every `motion` element that uses `whileInView` — the `amount: 0.1` means animation triggers when just 10% of the element is visible (instead of default 50%)
- Make sure no parent element has `overflow: hidden` that would prevent `IntersectionObserver` from detecting the child
- Remove `transform: translateZ(0)` from section wrappers on mobile — this can create new stacking contexts that break `whileInView`
- Reduce animation distances on mobile: use `y: 20` instead of `y: 60` for entrance animations so they feel appropriate for small screens
