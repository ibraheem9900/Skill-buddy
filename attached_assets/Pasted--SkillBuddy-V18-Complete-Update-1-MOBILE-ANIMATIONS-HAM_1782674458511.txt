## 1. MOBILE: ANIMATIONS + HAMBURGER + NAVBAR GAP (Critical)

**Animations not working on mobile:**
- Change ALL `whileInView` viewport settings to `{ once: true, amount: 0.05 }` — triggers when only 5% visible
- Remove `overflow: hidden` from ANY parent wrapper that contains animated children — this breaks IntersectionObserver
- Reduce all `y` animation values to `y: 20` maximum on mobile (not 60 or 80)
- Make sure no section has `transform: translateZ(0)` on mobile — creates new stacking context

**Hamburger not opening:**
- The hamburger button must have `type="button"` and `e.preventDefault()` + `e.stopPropagation()` in onClick
- It must NOT be inside any `<a>` or `<Link>` tag
- The drawer must render via `createPortal(content, document.body)` so it's outside all overflow containers
- Drawer z-index: 10001, backdrop z-index: 10000

**Navbar-hero gap on mobile:**
- Set `padding-top` on `<main>` equal to EXACTLY the navbar height (measure with `navRef.current.getBoundingClientRect().height`)
- Hero section itself: `padding-top: 0`, `margin-top: 0`
- Remove any extra spacing from first child of main on mobile
